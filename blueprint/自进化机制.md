## 二、回路 1：内容新鲜度监控

### 2.1 设计目标

自动检测数据过期、信息过时的笔记，生成更新提醒或自动提案。

### 2.2 新鲜度评分模型

每篇笔记根据以下维度计算"新鲜度分数"（满分 100）：

| 维度 | 权重 | 计算方式 |
|------|:----:|----------|
| 距最后更新时间 | 35% | `max(0, 1 - days/365) × 35` |
| 数据引用年份 | 30% | `max(0, 1 - (now - data_year)/2) × 30` |
| 外部事件冲击 | 20% | 关联政策/事件更新 → 扣分 |
| 用户反馈信号 | 15% | 纠错/过期标记 → 扣分 |

**评分区间**：

| 分数 | 状态 | 处理 |
|------|------|------|
| ≥ 80 | 🟢 新鲜 | 无需处理 |
| 50-79 | 🟡 注意 | 列入观察清单 |
| 30-49 | 🟠 陈旧 | 自动生成更新提醒，通知编辑 |
| < 30 | 🔴 过期 | 自动归档，可选"重新激活" |

### 2.3 自动更新提醒模板

```
┌─────────────────────────────────────────┐
│  📋 内容新鲜度提醒                       │
│                                         │
│  笔记：[[05-经济发展/经济概况2025]]       │
│  原因：数据引用年份为 2024，现已 2026    │
│  建议：更新 GDP、产业、财政等最新数据    │
│  关联：州统计局 2025 年公报已发布        │
│                                         │
│  [去更新]  [暂时忽略]  [标记为仍有效]     │
└─────────────────────────────────────────┘
```

### 2.4 外部事件触发器

当系统监测到以下事件时，自动触发关联笔记的新鲜度降级：

| 事件类型 | 监测源 | 关联笔记示例 |
|----------|--------|-------------|
| 年度统计公报发布 | 文山州统计局网站 | 经济数据类笔记 |
| 重大政策出台 | 州政府官网 | 政策与治理类笔记 |
| 景区评级变化 | 文旅部公告 | 文化旅游类笔记 |
| 行政区划调整 | 民政部公告 | 行政区划类笔记 |
| 重大项目开工/竣工 | 新闻监控 | 基础设施类笔记 |

## 三、回路 2：用户反馈回路

### 3.1 反馈类型

```
用户阅读笔记
    │
    ├── 👍 点赞 → 提升笔记权重 → 搜索排序提升
    ├── 🔗 纠错 → 生成纠错工单 → 编辑处理 → 更新笔记
    ├── ➕ 补充 → 生成补充提案 → 编辑审核 → 合并入笔记
    ├── ⏰ 标记过期 → 触发新鲜度降级
    └── 💬 评论 → 引发讨论 → 优秀评论可升级为"读者笔记"
```

### 3.2 纠错工作流

```
用户提交纠错
    │  格式：原文引用 + 错误说明 + 建议修正 + 信息来源
    ▼
┌──────────────────┐
│  自动分类         │
│  - 事实性错误     │
│  - 数据错误       │
│  - 表述不当       │
│  - 格式问题       │
└────────┬─────────┘
         ▼
┌──────────────────┐
│  优先级判定       │
│  数据错误 > 事实  │
│  错误 > 格式问题  │
└────────┬─────────┘
         ▼
┌──────────────────┐
│  分配给编辑       │
│  P0 (数据): 24h   │
│  P1 (事实): 72h   │
│  P2 (表述): 7天   │
└────────┬─────────┘
         ▼
┌──────────────────┐
│  处理结果通知      │
│  - 已修正         │
│  - 经核实无误     │
│  - 需更多信息     │
└──────────────────┘
```

### 3.3 反馈积分体系

激励用户参与反馈，积分可兑换权益：

| 行为 | 积分 |
|------|:----:|
| 有效纠错（被采纳） | +10 |
| 有效补充（被采纳） | +20 |
| 标记过期（被确认） | +5 |
| 优质评论（被精选） | +5 |
| 创建笔记（通过审核） | +50 |
| 笔记被选为"精选" | +100 |

### 6.2 质量评分模型

```
Quality Score = 
    0.25 × 结构完整度（frontmatter 字段完整性）
  + 0.25 × 链接健康度（wikilink 目标存在率）
  + 0.20 × 合规安全度（通过所有合规检查 = 满分）
  + 0.15 × 引用丰富度（平均每 500 字引用的来源数量）
  + 0.15 × 可读性（Flesch 可读指数 / AI 可读性评分）
```

### 6.3 质量排行榜与徽章

| 徽章 | 条件 | 效果 |
|------|------|------|

| ⭐ 编辑推荐 | 编辑手动标记 | 搜索结果首位 |

| 🔥 热门 | 周浏览量 ≥ 500 | "热"角标 |
| 🛡️ 权威来源 | 引用 ≥ 3 个官方数据源 | 可信度徽章 |

## 八、人在回路（Human-in-the-Loop）
