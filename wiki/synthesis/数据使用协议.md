## 一、数据引用白名单

### 1.1 分级使用权限

| 来源等级 | 置信度 | 引用权限 | 引用格式要求 |
|----------|--------|---------|------------|
| S（政府公报/白皮书） | 高 | **可直接引用** | `[来源: {具体公报名称}, {发布机构}]` |
| A（政府官网发布） | 高 | **可直接引用** | `[来源: {发布机构}, {发布日期}]` |
| B（央媒/省媒） | 中 | **限制引用** | 必须加"据{媒体}报道"，不得作为唯一论据 |
| B（单一来源） | 中 | **需标注** | `⚠️ 仅据{来源}单源报道，尚未完成交叉验证` |
| C（网站转载/行业网） | 低 | **禁止引用** | — |
| D（无来源/推测） | 不可用 | **绝对禁止** | — |

### 1.2 白名单例外机制

以下情况虽属B级来源，允许**有条件放宽**至可引用：

| 例外场景 | 条件 | 引用格式 |
|----------|------|---------|
| 央媒对政府官员的专访引述 | 专访原文可查，被采访人身份可确认 | `据{媒体}对{职务}{姓名}的采访` |
| 省媒发布的政府工作会议通稿 | 通稿注明"通讯员"来自政府部门 | `据{媒体}从{政府部门}获悉` |

## 二、数据演化规则

### 2.1 派生数据的定义

满足以下任一条件即为"派生数据"，**禁止作为事实陈述**：

1. 由两个或多个原始数据通过数学计算得出（如"年省6亿元"由电价差×用电量推算）
2. 由原始数据通过因果关系推断得出（如"铝土矿→氧化铝增值约3倍"）
3. 使用行业经验系数/类比推算得出（如"吨铝成本低2000元"链式派生）

### 2.2 派生数据标注规范

派生数据在文章中引用时必须同时标注：

| 标注要素 | 说明 | 示例 |
|----------|------|------|
| **测算标签** | 明确标注"测算"字样 | `(测算)` |
| **计算逻辑** | 用脚注或括号说明推算公式 | `(测算: 电价差0.15元/kWh × 年用电40亿kWh)` |
| **前提假设** | 说明计算依赖的关键假设 | `假设: 绿电直供覆盖全部电解铝用电需求` |
| **不确定性** | 如数据缺口较大，标注"±"范围 | `年省成本约6±1亿元` |

### 2.3 链式派生规则（重要）

链式派生（由 A 推 B、B 推 C、C 推 D）的数据禁止使用。每个派生数据**最多1次计算跳转**。

示例：
- ✅ 允许：电价差 × 用电量 → 年省成本
- ❌ 不允许：吨铝成本低 → 综合成本优势 → 竞争力提升 → 市场份额增长（4次跳转）

## 三、数据时效窗口

### 3.1 动态数据分类

| 数据类型 | 时效窗口 | 过期标记动作 | 示例 |
|----------|---------|------------|------|
| GDP/产值/收入 | 12个月 | 自动标记"可能已过期" | 2024年GDP用于2026年文章中 |
| 人口统计数据 | 12个月 | 自动标记"可能已过期" | 2024年常住人口 |
| 产能/产量 | 6个月 | 自动标记"请核实最新数据" | 电解铝产能343万吨(2023年数据) |
| 企业名单/项目清单 | 6个月 | 自动标记"请核实" | 在建项目清单 |
| 汇率/价格 | 3个月 | 自动标记"价格已变动" | 三七市场价格 |
| 规划/目标 | 12个月 | 标注"根据{X年}规划" | 2030年产值目标1800亿 |

### 3.2 时间窗口计算基准

以**当前日期（2026-05-25）** 为基准，向前推算：

- 2025年(含以后)的数据：**有效**
- 2024年数据：GDP/人口/旅游类 **已过期（>12个月）**，其他类需判断
- 2023年及以前数据：**已过期**，必须标注年份且不单独作为结论依据

### 3.3 过期数据使用限制

| 过期时长 | 使用限制 |
|----------|---------|
| 过期 ≤ 6个月 | 可引用，须标注数据年份 |
| 过期 6-12个月 | 可引用，须标注"截至{X年X月}" |
| 过期 > 12个月 | 仅作为历史对比/趋势分析引用，禁止作为"当前"状态描述 |

## 四、矛盾仲裁

### 4.1 权威层级

当同一数据存在多个来源且数值矛盾时，按以下优先级裁决：

| 优先级 | 来源类型 | 说明 |
|--------|---------|------|
| 1（最高） | 文山州统计公报 | 法定权威统计发布 |
| 2 | 云南省统计年鉴 | 省级权威 |
| 3 | 国家统计局发布 | 垂直系统最高权威，但可能粒度不够 |
| 4 | 政府工作报告 | 注意：报告中的"预计""目标"类数据低于统计公报事实数据 |
| 5 | 主管部门公报（能源局等） | 行业数据以行业主管为权威 |
| 6 | 央媒（人民日报/新华社） | 仅在前5级无数据时参照 |

### 4.2 已知矛盾项及裁决

当前知识库中经矛盾检测报告确认的矛盾项，裁决如下：

| 矛盾项 | 冲突值 | 裁决结果 | 裁决依据 |
|--------|--------|---------|---------|
| 常住人口 | 337.2万 ↔ 335.4万 | **335.4万** | 2025年统计公报更新 |
| 城镇化率 | 40.57% ↔ 41.19% | **41.19%** | 2025年统计公报 |
| 规上工业增速 | 17.5% ↔ 12.9% | **12.9%** | 需核实统计口径（累计vs当月，营收2000万以上vs全部） |
| 三七产值 | 500亿(目标) ↔ 456亿(现状) | **456亿** | 500亿为规划目标，456亿为2025年实际值 |
| 旅游收入 | 541.48亿 ↔ 425.7亿 | 两值口径不同 | 541.48亿为"旅游收入"，425.7亿为"旅游花费"，口径差异待确认 |
| 清洁能源99.8% | 装机占比 vs 发电量占比 | **装机占比** | 已确认非发电量占比，引用时必须注明"装机占比" |

### 4.3 矛盾项处理流程

```
发现矛盾 → 检查是否为同一口径 → 
  同口径 → 按权威层级裁决 → 低权威数据标记"存疑"
  不同口径 → 标注两个值的口径差异 → 避免混用
```

## 六、创作前核验工作流

### 6.1 标准流程

```
写作任务启动
    │
    ▼
① 从「数据资产登记表」拉取相关数据点
    │ 匹配条件：写作主题关键词 ↔ 登记表数据分类
    │
    ▼
② 按置信度分级筛选
    │ 高置信度 → 白名单直接引用
    │ 中置信度 → 标注限定语后引用
    │ 低/待核验 → 排除
    │
    ▼
③ 检查时效性
    │ 过期数据 → 标记年份，降级为历史对比
    │
    ▼
④ 检查矛盾项
    │ 已知矛盾 → 按裁决结果引用
    │
    ▼
⑤ 生成「本次写作数据清单」
    │ 含：数据ID、值、来源等级、引用格式
    │
    ▼
⑥ 写作完成后 → 自动生成文末数据溯源表
```

### 6.2 快捷核验清单

| 检查项 | 通过标准 |
|--------|---------|
| 所有数值型数据均有数据ID对应 | ✅ |
| 无C/D级来源数据出现在正文中 | ✅ |
| 所有派生数据均已标注"测算"及计算逻辑 | ✅ |
| 时效性检查已完成 | ✅ |
| 已知矛盾项已按裁决结果处理 | ✅ |
| 文末附数据溯源表 | ✅ |

## 七、违规处理

### 7.1 违规等级

| 等级 | 行为 | 后果 |
|------|------|------|
| 轻微 | 引用过期但仍在12个月内的数据，已标注年份 | 提醒更新 |
| 中等 | 引用B级来源未加限定语 | 必须修订 |
| 严重 | 引用C/D级来源数据 | 该数据点从产出物中移除 |
| 严重 | 派生数据标注为事实陈述 | 该段落需重写 |

### 7.2 本协议约束范围

本协议对所有基于 D:\WenShanKB 知识库生成的产出物（文章、报告、宣传材料等）具有同等约束力。
